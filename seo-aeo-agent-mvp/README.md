# SEO/AEO Agent MVP

Рабочий MVP-воркфлоу для SEO/AEO-пайплайна без 11 отдельных агентов.

В проекте используется компактная схема из 5 модулей:

1. `CatalogAgent` — загружает сущности и связи.
2. `SearchUnitBuilder` — строит валидные SEO-единицы.
3. `QueryBuilder + QueryFilter` — генерирует и чистит запросы.
4. `FrequencyCollector` — собирает частотность (в демо через mock-клиент).
5. `BriefAgent + ContentGenerator` — собирают brief и генерируют SEO/AEO-блоки.

## Что это делает

- создает SQLite БД;
- загружает демо-каталог;
- строит `search_unit`;
- генерирует кандидаты запросов;
- фильтрует мусор и услуги;
- собирает частотность;
- собирает brief;
- генерирует SEO/AEO тексты;
- сохраняет результаты в `out/summary.json`.

## Запуск

```bash
python main.py
```

После запуска появятся:

- `seo_agent.db` — SQLite база;
- `out/summary.json` — сводный JSON;
- `out/generated_content.json` — тексты по unit'ам.

## Почему не 11 агентов

Для MVP достаточно 5 сервисов внутри одного оркестратора. Это проще сопровождать, дешевле и надежнее.

## Как расширять

- заменить `MockWordstatClient` на реальный клиент Wordstat API;
- добавить PostgreSQL вместо SQLite;
- заменить шаблонный генератор на LLM;
- добавить метрики страниц и повторный цикл обновления;
- вынести модули в очереди и scheduler.
